## 品質管理概述
- 品質管理 : 監控製造過程中的變數(人/機/料/法)進而掌握品質變化狀況.
- 4M : 人/機/料/法
```
Man:作業者
Machine:機台
Material:材料
Method:作業方法

```

## 如何監控品質
- 最適品質=最適合客戶使用之產品=滿足客戶需求  
=> 規格=客戶需求  

- 監控品質=監控產品是否於規格內

- 品質可分為兩大類:
```
  A.記數值-計算單位僅能以整數方式計算之
           單位值,如:1個/1瓶/1顆…
  B.記量值-計算單位可明確表示長/寬/高
           /重量…等之單位值
           如:2.54cm/3.67mg…  
```
一般而言,產品品質的狀態多以數據方式進行呈現,  
所以,以統計方式進行監控及分析為較常用之品質監控方式

### 直方圖-可看出數據離散程度 藉以判定製程能力
分析步驟:
```
1.資料收集—收集所需data(data數量需超過32筆資料)
2.依收集的data區分群組數目;一般而言,區分之群組數需依data數量進行區分,區分方式為data量開根號
ex:data收集數量共50筆,則可區分為 7筆組距
3.將所收集data依組距分類依序置入,將各組距data數量依表格方式畫出,即可得到直方圖
```
#### EX
```
SUNNY欲進行NFXXXX品目機台打拔狀況能力的評估,因此希望藉由直方圖呈現得知數據分佈狀況,進而得知現有機台打拔能力分佈狀況.

STEP 1:共收集DATA數為100筆資料,資料分佈如下
       =>MAX=0.80   MIN=0.59   AVG=0.668 
```
![image](https://user-images.githubusercontent.com/79491888/177684033-c438afd3-6ec8-419c-8566-8d1ce11688ad.png)
```
STEP2:決定區間數為何.
        由於DATA共100個,故區間數為100的開根號=10組

STEP3:求出區間的寬幅
```
![image](https://user-images.githubusercontent.com/79491888/177684235-d636e31a-11a8-4541-adac-ecd0c425a647.png)

```
  由於最小測量值為0.01,故取區間寬幅為0.02

```
```
STEP4:決定區間上下界值及中間值
          第1區間下限值=最小值-(最小量測值/2)
                                =0.59-(0.01/2)=0.585
          第1區間上限值=第1區間下限值+區間寬幅
                                =0.585+0.02=0.605
          第1區間中間值=(第1區間下限值+第1區間上限值)/2
                                =(0.585+0.605)/2=0.595

STEP5:標示出各區間下限值/上限值/中間值
```
![image](https://user-images.githubusercontent.com/79491888/177684540-ebf1a64c-f57a-4c82-80b4-1862ebef207c.png)

```
STEP6 :將數據分類後依表格區間依序劃記填入,並計算其次數
```
![image](https://user-images.githubusercontent.com/79491888/177684648-54c90cce-2902-40de-8f6c-3c95ba18c002.png)
```
STEP7 :將數據分類後依表格區間依序劃記填入,並計算其次數
```
![image](https://user-images.githubusercontent.com/79491888/177684716-ecdcf161-0323-4b18-b1dc-5b3ca7c235f8.png)
```
STEP8:判讀圖形
判讀的重要在於直方圖的特徵：
   [1] 數據分佈的情形
   [2] 數據的中心位置
   [3] 數距離散的程度
   [4] 數據和規格的關係

依完成的直方圖在全面檢討下，做出了下面結論：
   數據的分佈情形幾乎成鐘形，數據的中心也差不多在中央部份（x=0.668),顯示製程的準確度高

```
