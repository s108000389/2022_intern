## 品質管理概述
- 品質管理 : 監控製造過程中的變數(人/機/料/法)進而掌握品質變化狀況.
- 4M : 人/機/料/法
```
Man:作業者
Machine:機台
Material:材料
Method:作業方法
```
5M : 人/機/料/法/測量
```
Measurement:測量
```

## 如何監控品質
- 最適品質=最適合客戶使用之產品=滿足客戶需求  
=> 規格=客戶需求  

- 監控品質=監控產品是否於規格內

- 品質可分為兩大類:
```
  A.記數值-計算單位僅能以整數方式計算之
           單位值,如:1個/1瓶/1顆…
  B.記量值-計算單位可明確表示長/寬/高
           /重量…等之單位值
           如:2.54cm/3.67mg…  
```
一般而言,產品品質的狀態多以數據方式進行呈現,  
所以,以統計方式進行監控及分析為較常用之品質監控方式

### 直方圖-可看出數據離散程度藉以判定製程能力
分析步驟:

1.資料收集—收集所需data(data數量需超過32筆資料)  
2.依收集的data區分群組數目;一般而言,區分之群組數需依data數量進行區分,區分方式為data量開根號  
ex:data收集數量共50筆,則可區分為 7筆組距  
3.將所收集data依組距分類依序置入,將各組距data數量依表格方式畫出,即可得到直方圖  

#### EX
```
SUNNY欲進行NFXXXX品目機台打拔狀況能力的評估,因此希望藉由直方圖呈現得知數據分佈狀況,進而得知現有機台打拔能力分佈狀況.

STEP 1:共收集DATA數為100筆資料,資料分佈如下
       =>MAX=0.80   MIN=0.59   AVG=0.668 
```
![image](https://user-images.githubusercontent.com/79491888/177684033-c438afd3-6ec8-419c-8566-8d1ce11688ad.png)
```
STEP2:決定區間數為何.
        由於DATA共100個,故區間數為100的開根號=10組

STEP3:求出區間的寬幅
```
![image](https://user-images.githubusercontent.com/79491888/177684235-d636e31a-11a8-4541-adac-ecd0c425a647.png)

```
  由於最小測量值為0.01,故取區間寬幅為0.02

```

### 柏拉圖-為80/20法則理論應用藉以找出重點項目(重點因素)
#### 柏拉圖
柏拉圖是指，將不良數目與損失金額依現象及原因等的分類項目分
類，再依據大小順序用柱狀圖表示之，其累計和則用推移圖表示的圖  
```
1.著重於改善效果大且重要的問題，並改善之。
2.在所有許多問題中，追究對結果造成最大影響的原因，並採取因應對策。

柏拉圖是：實行這些重點指向之際，幫助找出重點問題所在的重要手法。
```
#### EX:4月份鍍金工程共製作了20個品目,為改善該工程不良現況,故必須找出重點品目進行對策,以快速降低不良率.

### 魚骨圖-要因解析專用以利於問題分析

所謂魚骨圖=特性要因圖，是指整理特性（結果）以及可能對之造成  
影響之要因（原因）的關連性，有體系的畫成像魚骨一般的圖。  
它是管理、改善工作不可欠缺的，是重要的QC七大手法之一 。  

<img src="https://user-images.githubusercontent.com/79491888/177688320-7c5d8107-37b7-4a7c-9ee9-701c255096bd.png" width="500px">


### 散佈圖-為找出兩組數據間的相關聯程度適用
所謂散佈圖，是將成對的兩個特性值設定於縱軸、橫軸後，將各組  
之數據以點記之方式所作成之圖，用以觀察此兩個特性值是否呈現  
無關係狀態變化，亦或是互相交互影響變化著。  

<img src="https://user-images.githubusercontent.com/79491888/177688668-7079c840-eb0f-408b-ac5b-259f7a52196d.png" width="300px">


### 管制圖-藉以管理資料走向趨勢作為管理推移依據
所謂管制圖，是為了調查工程（製程）是否處於安定狀態，或是使工程維持於規格內，所使用的圖表。
![image](https://user-images.githubusercontent.com/79491888/178169398-4afba3af-ce25-4c53-add6-570f3731a082.png)
- 管制圖中一般會有規格上/下限,規格中心及管制上/下限,其中規格上下限及中心值為固定值,但管理上/下限則會因實際量測平均值狀況而有所變異


### 層別法-常用於4M分析歸納藉以找出問題主因趨勢為何
將得到的數據資料和調查對象等，依作業員別、機器設備別、材料零件別、作業方法別等分類分級，利用分類分級，可以發現許多問題。

## SPC
SPC-為管制圖的運用主要用來監控製程的穩定性與否, 藉由每階段的紀錄及趨勢分析,得知製程的能力並進行及時的改善及評估.
- SPC中常用簡寫及公式
```
規格上限(USL)
規格下限(LSL)
規格中心(CL)=((USL+LSL)/2)
組距=T=(USL-LSL)
平均值=AVERAGE=X總合/個數
最大值(Maximum)=數據中之最大值
最小值(Minimum)=數據中之最小值
```

## QCC活動
所謂QCC活動,主要就是依照QC STORY的方式進行品質改善活動,藉由P->D->C->A的循環,將品質不斷往上提升.並藉由討論的過程中,不斷的集思廣益,可讓參與的眾人,由活動中增加對問題解析的能力  
PDCA（Plan-Do-Check-Act的簡稱）: 循環式品質管理，針對品質工作按規劃、執行、查核與行動來進行活動，以確保可靠度目標之達成，並進而促使品質持續改善。  
- 主題>現況掌握>要因分析>對策擬定>對策執行>效果評估>標準化  
### QCC活動--活動步驟
```
1.主題選定
2.活動計畫擬定
3.現況把握
4.目標設定
5.要因分析
6.對策擬定
7.對策實施
8.效果確認
9.標準化
10.活動檢討

```

## 5S
```
起源於日本的5S運動，原本的目的是要讓工作場所的工具擺放有序、
提升工作安全及效率，降低產品不良率，日本製造業因為推行成效良好，
使得日本商品成為世界頂尖的代名詞，於是就有跨國大企業將5S推行到其他各地。
```

- 何謂5S運動
若以羅馬拚音書寫，它的每個字都是以「S」為首字，因此稱為5S。其字面意義為:  
```
一、整理（Seili）：區分要與不要的東西，工作場所除了要的東西以外，一切都不可以放置。
二、整頓（Seiton）：任何人在想要什麼東西時，都可以隨時取到想要的東西。
三、清掃（Seisoa）：將看得到與看不到的工作場所清掃乾淨，保持整潔。
四、清潔（Seiketu）：貫徹整理、整頓、清掃的3S，而使同仁工作效率提升。
五、教養（Situke）：由內心發出養成遵守紀律，並且以正確的方法去做。
```

## 觀念-作業標準文件
作業標準書就是工作作業流程標準程序，製作作業標準書的目的，有以下幾點：  
- 一、將作業內容正確傳達給現場的作業人員
- 二、明確的表示作業方法，使作業能變成簡單化、標準化
- 三、工作職務內容交接最佳的工作移交說明書
- 四、現場指導員、監督人員容易進行工作確認管理
- 五、作業改善與製造新技巧的記錄
